# ADR 变更日志

## 2024-04-08

### 新增 ADR

- **ADR-006**: 命令系统设计

  - 支持跨平台的命令一致性
  - 包含危险命令的审批流程
  - 添加了命令系统的 Mermaid 架构图

- **ADR-007**: 会话管理系统
  - 定义了集中式会话管理器的架构
  - 支持跨平台的会话持久化
  - 包含会话生命周期和清理策略
  - 添加了会话生命周期的 Mermaid 状态图

- **ADR-008**: 技能系统架构
  - 定义了基于文件的技能系统
  - 支持技能的自动创建和改进
  - 包含技能验证和安全检查
  - 添加了技能生命周期的 Mermaid 状态图

## 变更类型说明

- **新增 ADR**: 新的架构决策记录
- **更新**: 对现有 ADR 的内容更新
- **弃用**: ADR 被标记为已弃用
- **替代**: ADR 被新的决策替代
- **增强**: 为 ADR 添加新的内容（如图表、示例）

## 贡献指南

### 如何添加新 ADR

1. 使用 ADR 模板创建新文件
2. 编号递增（如 ADR-009）
3. 提交 PR 并说明决策背景
4. 等待团队审查和批准

### 如何更新现有 ADR

1. 修改对应的 ADR 文件
2. 在变更日志中记录更新内容
3. 提交 PR 并说明更新原因
4. 等待团队审查和批准

### 如何弃用 ADR

1. 在 ADR 顶部添加状态标记：`## 状态 ❌ 已弃用`
2. 说明弃用原因和替代方案
3. 在变更日志中记录弃用信息
4. 交叉引用替代的 ADR
